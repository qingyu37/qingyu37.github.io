正则表达式

发表于 2023-04-20

在完成软件构造lab3的过程中，用到了正则表达式，为了更好地完成实验，对正则表达式以及java中正则表达式的使用进行复习并记录如下。

知识点总结

介绍

  1. 正则表达式是一种用于匹配和操作文本模式的工具。它可以用来搜索、替换和验证文本数据。

  2. 正则表达式由字符和特殊字符组成。字符可以匹配自身，而特殊字符用于表示模式中的特定规则。

  3. 常见的特殊字符包括：

    • .：匹配任意单个字符（除了换行符）。
    • *：匹配前面的字符零次或多次。
    • +：匹配前面的字符一次或多次。
    • ?：匹配前面的字符零次或一次。
    • ^：匹配字符串的开头。
    • $：匹配字符串的结尾。
    • \：转义字符，用于匹配特殊字符本身。

  4. 字符类用于匹配一组字符中的任意一个字符。使用方括号[]表示字符类，例如[abc]匹配字符”a”、”b”或”c”。

  5. 元字符用于表示一组字符的缩写形式，常见的元字符包括：

    • \d：匹配任意数字。
    • \w：匹配任意字母、数字或下划线。
    • \s：匹配任意空白字符（空格、制表符、换行符等）。
    • \D：匹配任意非数字字符。
    • \W：匹配任意非字母、数字或下划线字符。
    • \S：匹配任意非空白字符。

  6. 量词用于指定匹配的次数。常见的量词包括：

    • {n}：匹配前面的字符恰好n次。
    • {n,}：匹配前面的字符至少n次。
    • {n,m}：匹配前面的字符至少n次且不超过m次。
    • *、+、?都可以看作是特殊的量词，分别表示零次或多次、一次或多次、零次或一次。

  7. 分组和捕获允许将一部分模式分组并进行操作。使用小括号()表示分组，使用捕获组可以在匹配时提取分组的内容。

举例

以在软构实验中用到的一个捕获原子的轨道系统上电子数的正则表达式为例：

\s*NumberOfElectron\s*::=\s*(\d+\d+;)+\d+\d+
  1. \s* 表示匹配任意空格
  2. \s*NumberOfElectron\s*::= 表示开始读入电子数
  3. (\d+\d+;)+ 表示匹配轨道号及对应的电子数
  4. \d+\d+ 表示至少有一层轨道

本篇讲介绍系统层次的可复用性——框架。
Java 框架是开发人员在用 Java 编程语言来创建应用时，所使用的一系列预编写代码。

Java 框架是 Java 编程语言所特有的，是一个用于开发软件应用和 Java 程序的 Java 平台。

Java 框架由一系列可重用的预编写代码组成，它们起着模板的作用，开发人员可以根据需要通过填充自定义代码来创建应用。

框架创建后可反复使用，这样开发人员即可以在一定的结构上来编写应用，而无需从头开始手动创建。

Java 框架中可以包含预定义类（例如对象类别）和函数，用于处理、输入和管理硬件设备，以及与系统软件进行交互。当然，具体的框架内容要取决于框架的类型、Java 开发人员的技能水平、他们所要完成的工作以及自己的偏好。

为什么要使用框架

框架的优越性在于：

  1. 集中精力完成系统的业务逻辑设计；
  2. 使程序更容易扩展和维护，稳定性更强；
  3. 处理系统很多细节问题，比如安全性问题。

如何使用

  1. 导入相关的jar包
  2. 配置项目配置文件（比如web.xml），让项目用框架处理各种行为
  3. 添加框架配置文件（比如struts-config.xml），设定框架如何处理各种行为（比如接收到某个请求时，调用哪个类处理）
  4. 编写业务逻辑（继承、实现框架中的类、接口），完成具体处理操作。

主流框架

用得最多的Java三大框架是Struts、Spring和Hibernate ，简称SSH；14年以后逐步开始采用Spring、SpringMVC和MyBatis，简称SSM；而现在Java开发用得最多的框架其实是SpringBoot。

SSH的特点

SSH框架是以Struts框架进行MVC分离、控制业务跳转，同时使用Hibernate进行持久化，最后配合Spring的统一管理进行实现的开发框架。相对于传统的J2EE开发框架来讲性能相对较高、资源消耗相对较少。具体优势如下：

  • 框架进行了代码分层，将Java代码和HTML代码进行分层，大大降低了开发人员的技术难度，不需要同时精通Java后端知识和Web前端知识。
  • 由于使用了Struts框架进行了MVC分离，降低了各层代码间的耦合度。
  • 与Spring的AOP机制相互独立，互不影响。
  • 使用SSH框架开发项目，能够大大提高项目的开发效率与开发质量。
    良好的扩展性和维护性。

SSM框架的特点

SSM框架看着是Spring+SpringMVC+MyBatis三个框架整合而成的，其实SpringMVC属于Spring框架，所以相当于SSM框架就是Spring和Mybatis两个框架的整合运用

  • 对于MVC模式的实现，SpringMvc更加的轻量。
  • 由于SpringMvc本身属于Spring，因此与Spring的配合使用契合度更高。
  • Mybatis相对Hibernate更加轻量，同时将Sql和业务代码进行了很好的解耦。
  • MyBatis的学习门槛更低，相对于传统的JDBC差距不大，更好入门及掌握。

SpringBoot框架

SpringBoot框架是基于Spring4.0设计的，使用SpringBoot框架进行应用开发能够使用Spring框架所有优秀特性，同时还能够减少各种复杂的配置过程，降低各依赖包的冲突、增强系统的稳定性。我们可以将SpringBoot框架理解为我们使用SSM框架的一个脚手架框架使用，使用它能够非常方便的构建我们的应用。

  • 以非常方便和快速的构建Spring相关应用
  • 可以快速构建可执行的Jars和Wars文件
  • 内嵌了Servlet容器，如Tomcat、Jetty、Undertow
  • 使用SpringBoot框架开发项目，可以让Spring的配置操作几乎为零
  • SpringBoot框架提供了运行时监控功能
  • 框架极大的提高了项目开发和部署的效率

为重复使用此资源需花费多少代价？

  • 调整和扩展它的成本
  • 实例化它的成本
  • 更改与其交互的系统的其他部分的成本

可复用性意味着对构建，打包，分发，安装，配置，部署，维护和升级问题进行一些明确的管理。

具有高可复用性的软件资源应该：

  • 便携式和标准合规性
  • 通用和参数化
  • 模块化
  • 改变要求时的稳定性
  • 丰富的文档

Java作为一种面向对象编程语言，其核心思想之一就是“可复用性”，也就是说Java中的代码应该能够被多个程序或模块重复利用，从而减少代码的冗余，提高程序的可维护性和可扩展性。在这里主要介绍源代码和模块级别的复用。

  • 源代码级别复用
  • 模块级别的复用：类/抽象类/接口
  • 库级别的复用：API /包
  • 系统级别的复用：框架

1. 封装代码

封装是指隐藏实现细节，只暴露必要的接口，让外部调用者无法直接访问类的内部数据。这样做可以使代码更加安全和可靠，同时也可以减少代码耦合度，提高代码的可复用性。

例如，一个类可以通过提供公共方法来让外部调用者访问其私有变量。这样，如果需要修改类的实现细节，只需要修改对外暴露的方法而不需要修改所有依赖该类的代码。

2. 继承

继承是Java中实现代码可复用性的一种方式。通过继承，我们可以在已有的类基础上创建一个新的类，新的类将自动拥有基类的属性和方法。这样就可以避免重复编写相同的代码。同时，继承也可以帮助我们构建更加灵活的代码结构，使得程序具有更好的可扩展性。

例如，我们可以定义一个基类Animal，其中包含一些通用的属性和方法，然后定义一些具体的子类，如Dog、Cat等，这些子类将自动继承基类的属性和方法，从而避免了重复编写相同的代码。

3. 接口

接口也是Java中实现代码可复用性的一种方式。接口定义了一组规范，可以在多个类中实现，从而使得这些类能够共享相同的行为。接口可以帮助我们实现程序的多态性，使得程序具有更高的可扩展性和灵活性。

例如，我们可以定义一个接口Animal，其中包含一些通用的方法，然后让不同的类实现该接口，从而共享这些方法。这样，我们可以很容易地添加新的实现类，而不需要修改原有的代码。

4. 泛型

Java中的泛型是一种类型参数化机制，它可以让代码更加通用、安全和可读。在使用泛型时，我们可以将类型作为参数传递给类、接口、方法或其他类型，从而创建一个具有通用性的类或方法。使用泛型，可以减少代码中的类型转换，避免运行时错误，并提高代码的可读性和可维护性。

AF、RI和safety from rep-exposure

发表于 2023-03-28

在Java编程语言中，代码的可读性和可维护性是非常重要的。为了确保这些特性，Java提供了一些机制来确保代码的正确性和安全性。其中包括AF（Abstraction Function）、RI（Representation Invariant）和Safety from Rep Exposure。

抽象函数(AF)

首先，让我们来看看什么是抽象函数（Abstraction Function）。在Java中，抽象函数是一种将数据结构抽象化的方式，可以将其看作一个黑盒子。通过隐藏数据结构的内部细节，抽象函数使我们能够更好地管理代码，并且更容易修改和维护它。例如，在Java中，List接口可以被视为一种抽象函数，它定义了访问列表的方法，但并不关心它是如何实现的。

表示不变量(RI)

其次，我们来看看表示不变量（Representation Invariant）。在Java中，表示不变量是一种约束，确保对象的状态始终符合我们期望的条件。通常，它们是通过对类的成员变量和方法进行注释来定义的。例如，假设我们有一个叫做Person的类，它有一个成员变量叫做age。我们可以通过注释来表示age必须是一个正整数，即表示不变式为“age > 0”。

防止表示泄露

最后，让我们来看看防止表示泄露（Safety from Rep Exposure）。在Java中，表示泄露是一种很重要的安全问题，它可能导致对象状态的意外更改。为了防止这种情况发生，Java要求我们使用访问方法来访问对象的状态，而不是直接访问其成员变量。这意味着我们需要将类的成员变量设置为private，并提供公共方法来访问它们。例如，假设我们有一个叫做BankAccount的类，它有一个成员变量叫做balance。我们可以通过将balance设置为private，然后提供一个名为getBalance的公共方法来防止对balance的表示曝露。

综上所述，AF、RI和Safety from Rep Exposure是Java编程语言中的重要概念，它们可以帮助我们确保代码的正确性和安全性。通过使用抽象函数，我们可以将代码抽象化，从而更容易地管理和维护它。通过使用表示不变式，我们可以确保对象的状态始终符合我们期望的条件。通过使用Safety from Rep Exposure，我们可以防止表示曝露，从而保护对象的状态免受意外更改的影响。

equals and hashcode

发表于 2023-03-27 更新于 2023-03-28

#软件构造
在java中，我们经常需要判断两个对象是否相等，为了达到这个目的，往往需要重写equals方法和hashcode方法。

为什么

为什么要使用equals方法

在java中我们会使用 == 来对int、double等基本数据类型来进行相等判断，但是不能用 == 对引用数据类型进行相等判断，因为 == 在对引用数据类型判断时是比较两个对象的内存地址，无法通过判断对象内部的属性是否相等来判断两个对象是否等价，因而我们需要使用equals方法。

为什么要重写equals方法

java中的所有对象都有一个默认的祖宗——我是说默认的父类，也就是Object类，所以这些对象会继承它们祖宗的equals方法，让我们来看看Object类中是怎么样实现equals方法的。

是的，它是直接使用 == 来进行比较的，自然，这样是无法满足我们的需求的，所以我们需要在自己的每一个实现类中重写equals方法，以满足判断等价的需求。

为什么要重写hashcode方法

在你不会创建类对应的散列表的时候，重写hashcode方法与否其实是无伤大雅的，毕竟你基本是不需要调用这个方法的（但是可能会违反等价的对象必须具有相等的哈希码的要求）。
但是，如果需要创建类对应的散列表，比如自定义类作为hashSet，hashMap等的泛型时，就一定要重写hashcode方法了。
hashMap的put方法，使用hashCode()用key作为参数计算出hash值，然后进行比较，如果相同，再通过equals()比较key值是否相同，如果相同，返回同一个对象。
而hashSet中要使用hashMap中的put方法。
所以这时候就需要把equals方法和hashcode方法全部重写成符合规范的样子了，否则hashSet和hashMap就可能会出现诸如出现重复元素等的问题。

怎么做

怎么重写equals方法

equals方法返回true的条件是要满足等价，也就是自反、传递、对称。

 自反：一个对象对自身调用equals方法要得到一个true
 传递：对象A和对象B等价，B和C等价，那么A和C也等价
 对称：A.equals(B) == true，那么A和B等价

此外，还要满足不会对对象作出改变，一个对象对另一个对象多次调用equals方法应得到相同的结果。

从而给出一般的equals方法写法：

  1. 显示参数命名为 otherObject，稍后会将它转换成另一个叫做 other 的变量。(名字怎样都好，没什么所谓的)。
  2. 判断比较的两个对象引用是否相等，如果引用相等那么表示是同一个对象，那么当然相等。
  3. 如果 otherObject 为 null，直接返回false，表示不相等。
  4. 比较 this 和 otherObject 是否是同一个类：如果 equals 的语义在每个子类中有所改变，就使用 getClass 检测；如果所有的子类都有统一的定义，那么使用 instanceof 检测。
  5. 将 otherObject 转换成对应的类类型变量。
  6. 最后对对象的属性进行比较。使用 == 比较基本类型，使用 equals 比较对象。如果都相等则返回true，否则返回false。注意如果是在子类中定义equals，则要包含 super.equals(other)。

给出两个例子，一个是java自带的包装类String中重写的equals方法，另一个是自己的实现类中重写的equals方法。

怎么重写hashcode方法

按照hashCode方法的规约重写即可

  1. 如果两个对象通过equals方法比较是相等的，那么它们的hashCode方法结果值也是相等的。
  2. 如果两个对象通过equals方法比较是不相等的，那么不要求它们的hashCode方法结果值是相等的。
  3. 当在一个应用程序执行过程中， 如果equals方法比较中没有修改任何信息，那么在同一个对象上重复调用hashCode方法时，它必须始终返回相同的值。但如果从一个应用程序到了另一个应用程序，两个应用程序汇中调用hashCode方法的返回值可以是不一致的。

总结

在完成软件构造lab1的过程中，遇到的最大的困难是最小凸包集问题，通过查资料学习了这个问题的几种算法，在这里记录一下。

问题描述

用直线段顺序连接平面上的点，P1，…，Pn构成的封闭曲线称为一个多边形，Pi称为多边形的顶点，线段PiPi+1称为多边形的边。边中任意不相邻的顶点之间的连线称为多边形的弦。如果多边形的任意一条弦均位于多边形内部，则称该多边形是凸多边形。平面上n个点P1，…，Pn的凸包是使得P1，…，Pn位于其边界或内部的最小凸多边形。
凸包问题：
输入： 平面上n个点构成的集合S={(xi,yi)|xi,yi∈R,1≤i≤n}
输出： 以逆时针顺序输出S的凸包顶点

求解算法

1.暴力算法

基本思想

先用排除法确定凸包的顶点，也就是逐步删除不是凸包顶点的输入点，然后按逆时针顺序输入这些顶点即可。
使用两点确定一条直线，如果剩余的其他点都在这条直线的同一侧，则这两个点是凸包上的点，否则就不是。
至于如何判断剩余的其他点是否在这条直线的同一侧，可以使用叉乘的方法。

2.Graham扫描算法

基本思想

首先找到纵坐标最小的点，这个点肯定是凸包中的点，然后从这个点开始按极角扫描所有的输入点，并判断是否在凸包中，直至扫描到左下侧。

算法步骤

  1. 把所有点放在二维坐标系中，则纵坐标最小的点一定是凸包上的点，记为P0
  2. 计算各个点相对于P 0 P 0P0的幅角 α ，按从小到大的顺序对各个点排序。当 α 相同时，距离P0比较近的排在前面。我们由几何知识可以知道，结果中第一个点P1和最后一个点一定是凸包上的点。
  3. 以上，我们已经知道了凸包上的第一个点P0和第二个点P1，我们把它们放在栈里面。现在从步骤2求得的那个结果里，把P1后面的那个点拿出来做当前点，即P2。接下来开始找第三个点。
  4. 连接栈最上面的两个元素，得到直线L。看当前点是在直线L的右边还是左边。如果在直线的右边就执行步骤5；如果在直线上，或者在直线的左边就执行步骤6。
  5. 如果在右边，则栈顶的那个元素不是凸包上的点，把栈顶元素出栈。执行步骤4。
  6. 当前点是凸包上的点，把它压入栈，执行步骤7。
  7. 检查当前的点是不是步骤2那个结果的最后一个元素。是最后一个元素的话就结束。如果不是的话就把当前点后面那个点做当前点，返回步骤4。
  8. 最后，栈中的元素就是凸包上的点了。
