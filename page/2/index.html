java中的断言

发表于 2023-05-22 更新于 2023-05-24

介绍

在软件开发中，调试和测试是不可或缺的步骤。为了帮助开发人员在代码中捕获潜在的错误和异常，Java引入了断言（assertions）机制。断言是一种声明，用于在代码中插入检查点，以确保某个条件为真。本文将深入探讨Java中的断言机制，包括其语法、工作原理和最佳实践。

启用和禁用

默认情况下，Java虚拟机在执行代码时将忽略断言。为了启用断言，我们可以在启动Java程序时使用命令行选项-ea或-enableassertions。例如：

工作原理

当启用断言时，Java虚拟机将在执行代码中的断言语句时进行检查。如果断言失败（即条件为假），则会抛出AssertionError异常。这个异常可以被捕获和处理，或者在没有处理的情况下导致程序终止。

使用注意

  • 断言主要用于调试和开发阶段，不应在生产环境中过度使用。
  • 断言应该用于检查不应该发生的情况，而不是用于处理可能发生的错误。
  • 断言的条件应该是纯粹的布尔表达式，避免在断言中执行复杂的操作。
  • 断言不应该改变程序的状态或产生副作用。
  • 断言的信息应该清晰明了，方便识别断言失败的原因。

与异常处理的区别

断言与异常处理有所不同。断言是用于调试目的，用于捕获程序中的逻辑错误和不一致性。异常处理则用于处理运行时异常和错误条件。断言应该关注于程序的正确性和一致性，而异常处理则关注于程序的可靠性和健壮性。

总结

断言是Java中一个有用的调试工具，可以帮助开发人员捕获潜在的错误和异常。通过合理使用断言，开发人员可以更早地发现和解决问题，提高代码的质量和可靠性。然而，断言应该谨慎使用，并在生产环境中进行适当的禁用，以避免不必要的性能开销。

设计模式总结

发表于 2023-05-11

前两天终于做完软构lab3了，接下来对java中的一些设计模式做一个简单的总结。
首先，同样推荐一个b站up主@林斯基JI，它的设计模式讲的挺好的，距离生动形象。

  1. 工厂模式（Factory Pattern）： 工厂模式用于创建对象，将对象的创建逻辑与客户端代码分离。它通过一个工厂类来创建对象，客户端只需调用工厂类的方法即可获取所需的对象。工厂模式可以根据条件返回不同的对象实例，提供了灵活性和可扩展性。

  2. 观察者模式（Observer Pattern）： 观察者模式定义了对象之间的一对多依赖关系，当一个对象的状态发生变化时，其依赖的对象将自动得到通知并更新。在Java中，观察者模式可以使用java.util.Observer接口和java.util.Observable类实现。

  3. 装饰器模式（Decorator Pattern）： 装饰器模式允许动态地向对象添加额外的功能，而不需要修改其原始类。它通过将对象包装在装饰器类中，逐层叠加功能来实现。在Java中，可以使用继承和接口来实现装饰器模式。

  4. 策略模式（Strategy Pattern）： 策略模式定义了一组算法，将其封装成独立的策略类，并使它们可以互相替换。客户端可以根据需求选择不同的策略，而不需要了解具体的算法实现细节。在Java中，策略模式可以通过接口和多态性来实现。

  5. 适配器模式（Adapter Pattern）： 适配器模式用于将一个类的接口转换成客户端所期望的另一个接口。它使得原本不兼容的类可以一起工作。适配器模式可以通过继承或组合来实现适配器类。

  6. 模板方法模式（Template Method Pattern）： 模板方法模式定义了一个算法的骨架，将某些步骤的具体实现延迟到子类中。它提供了一个模板方法来定义算法的框架，而具体的步骤实现由子类决定。在Java中，可以使用抽象类和钩子方法来实现模板方法模式。

委派

发表于 2023-05-10 更新于 2023-05-11

介绍

在Java开发中，设计模式是一种重要的概念，它们提供了一套经过验证的解决方案，用于解决常见的软件设计问题。委派模式是一种常见的设计模式，它允许对象通过将任务委派给其他对象来实现代码重用和灵活性。本文将深入探讨Java中的委派模式，介绍它的概念、工作原理以及常见的使用场景。

委派模式是一种对象行为模式，其中一个对象（称为委派者）将任务委派给其他对象（称为被委派者）来执行。委派者负责分发任务，并且不处理任务的具体实现细节。被委派者实际上执行任务，并返回结果给委派者。委派模式可以在运行时动态地选择被委派者，从而实现灵活的任务处理。

工作原理

当委派者接收到任务时，它根据一定的条件或策略选择合适的被委派者来执行任务。委派者本身不处理任务，它只负责分派和协调工作。被委派者实际上执行任务，并返回结果给委派者，然后委派者将结果返回给调用者。

使用场景

  • Strategy设计模式：委派模式可以用于实现任务调度器，其中委派者根据不同的条件将任务委派给不同的处理器。
  • visitor设计模式：委派模式可以用于处理事件，其中委派者将事件委派给适当的处理器来执行相应的操作。
  • plugin设计模式：委派模式可以用于插件系统，其中委派者负责将任务委派给适当的插件来执行特定的功能。

实例

一个常见的使用委派模式的实例是Java Servlet中的请求调度器（RequestDispatcher）。Servlet是Java Web应用程序的基本组件，用于处理客户端的HTTP请求并生成响应。

在Servlet中，请求调度器充当委派者的角色，负责将请求转发给适当的处理器（被委派者）进行处理。请求调度器根据请求的特征或路由规则，将请求委派给正确的Servlet或其他处理器进行处理。这样，请求调度器实现了请求的分发和协调，而不需要处理请求的具体实现细节。

通过请求调度器的使用，Servlet实现了请求的分派和协调，将具体的请求处理委派给适当的处理器。这种委派模式的应用可以使代码更加模块化和可扩展，提高了Web应用程序的灵活性和可维护性。

与继承的比较

委派模式和继承都可以实现代码重用，但它们的目的和机制有所不同。委派模式强调任务的分派和协调，它将任务委派给其他对象来执行。而继承是一种面向对象编程的基本概念，它允许一个类继承另一个类的属性和方法。继承用于实现类之间的层次关系和共享代码，但它通常会导致紧耦合的设计，难以灵活地改变行为。

总结

委派模式是一种在Java开发中常见且有用的设计模式，它提供了一种灵活的方式来实现代码的重用和任务的分派。通过委派模式，我们可以将任务委派给不同的对象，并实现代码的解耦和灵活性。了解委派模式的概念和使用场景，可以帮助我们在软件设计中更好地选择合适的模式，并编写可维护和可扩展的代码。

Liskov替换原则

发表于 2023-05-03

Liskov替换原则（Liskov Substitution Principle，LSP）是面向对象设计中的重要原则之一。本文将介绍Liskov替换原则在Java中的应用，以及如何遵循该原则来构建高质量的代码。

简介

Liskov替换原则是由计算机科学家Barbara Liskov提出的，它是SOLID原则中的一部分。该原则表述为：“如果S是T的子类型，那么在程序中任何使用T的地方都可以用S替换，而不会导致程序的行为发生变化。”简而言之，LSP要求子类型必须能够替代父类型，而不会引入意外的行为变化。

接口与实现的一致性

在Java中，接口是一种常见的定义抽象行为的方式。遵循LSP意味着实现接口的具体类应该能够完全替代接口本身，而不会破坏系统的正确性。具体类应该遵循接口定义的约定和行为，不得修改接口方法的前置条件、后置条件和约定。

继承关系的合理性

在Java中，继承是一种重要的代码复用和扩展机制。但是，滥用继承关系可能导致违反LSP。子类应该在继承父类的同时保持一致性和合理性。子类可以扩展父类的行为，但不应该修改父类的约定或者破坏原有的行为。违反LSP的继承关系可能导致意外的错误和行为不一致。

参数和返回类型的一致性

方法的参数和返回类型也需要符合LSP。子类的方法应该能够接受父类方法所接受的参数类型，并且返回类型应该是父类方法返回类型的子类型或者相等类型。这确保了代码在使用多态时的正确性。

设计模式与LSP

LSP对于许多常见的设计模式也有指导意义。例如，策略模式、模板方法模式等都要求子类能够替代父类并保持行为的一致性。遵循LSP可以增强设计模式的可维护性和可扩展性。

介绍

装饰器模式是一种通过包装对象来扩展其功能的设计模式。它通过创建一个包装器类，该类包含与原始对象相同的接口，并在内部维护一个原始对象的引用。装饰器类可以在调用原始对象之前或之后执行附加的操作，从而实现对原始对象功能的透明扩展。
本文将介绍Java中的装饰器模式，以及如何使用它来增添代码的灵活性和可扩展性。

组成

  1. 抽象组件（Component）：定义了原始对象和装饰器对象的公共接口，确保它们都可以被替代使用。
  2. 具体组件（ConcreteComponent）：实现了抽象组件接口，是被装饰的原始对象。
  3. 抽象装饰器（Decorator）：继承了抽象组件，并持有一个抽象组件的引用，以便可以包装原始对象。
  4. 具体装饰器（ConcreteDecorator）：扩展了抽象装饰器，实现了装饰器的具体功能，并在调用原始对象之前或之后执行额外的操作。

正则表达式

发表于 2023-04-20

在完成软件构造lab3的过程中，用到了正则表达式，为了更好地完成实验，对正则表达式以及java中正则表达式的使用进行复习并记录如下。

知识点总结

介绍

  1. 正则表达式是一种用于匹配和操作文本模式的工具。它可以用来搜索、替换和验证文本数据。

  2. 正则表达式由字符和特殊字符组成。字符可以匹配自身，而特殊字符用于表示模式中的特定规则。

  3. 常见的特殊字符包括：

    • .：匹配任意单个字符（除了换行符）。
    • *：匹配前面的字符零次或多次。
    • +：匹配前面的字符一次或多次。
    • ?：匹配前面的字符零次或一次。
    • ^：匹配字符串的开头。
    • $：匹配字符串的结尾。
    • \：转义字符，用于匹配特殊字符本身。

  4. 字符类用于匹配一组字符中的任意一个字符。使用方括号[]表示字符类，例如[abc]匹配字符”a”、”b”或”c”。

  5. 元字符用于表示一组字符的缩写形式，常见的元字符包括：

    • \d：匹配任意数字。
    • \w：匹配任意字母、数字或下划线。
    • \s：匹配任意空白字符（空格、制表符、换行符等）。
    • \D：匹配任意非数字字符。
    • \W：匹配任意非字母、数字或下划线字符。
    • \S：匹配任意非空白字符。

  6. 量词用于指定匹配的次数。常见的量词包括：

    • {n}：匹配前面的字符恰好n次。
    • {n,}：匹配前面的字符至少n次。
    • {n,m}：匹配前面的字符至少n次且不超过m次。
    • *、+、?都可以看作是特殊的量词，分别表示零次或多次、一次或多次、零次或一次。

  7. 分组和捕获允许将一部分模式分组并进行操作。使用小括号()表示分组，使用捕获组可以在匹配时提取分组的内容。

举例

以在软构实验中用到的一个捕获原子的轨道系统上电子数的正则表达式为例：

\s*NumberOfElectron\s*::=\s*(\d+\d+;)+\d+\d+
  1. \s* 表示匹配任意空格
  2. \s*NumberOfElectron\s*::= 表示开始读入电子数
  3. (\d+\d+;)+ 表示匹配轨道号及对应的电子数
  4. \d+\d+ 表示至少有一层轨道

    • start()：返回上一次匹配的子字符串的起始索引。
    • end()：返回上一次匹配的子字符串的结束索引。

代码的可复用性(3)

发表于 2023-04-08

本篇讲介绍系统层次的可复用性——框架。
Java 框架是开发人员在用 Java 编程语言来创建应用时，所使用的一系列预编写代码。

Java 框架是 Java 编程语言所特有的，是一个用于开发软件应用和 Java 程序的 Java 平台。

Java 框架由一系列可重用的预编写代码组成，它们起着模板的作用，开发人员可以根据需要通过填充自定义代码来创建应用。

框架创建后可反复使用，这样开发人员即可以在一定的结构上来编写应用，而无需从头开始手动创建。

Java 框架中可以包含预定义类（例如对象类别）和函数，用于处理、输入和管理硬件设备，以及与系统软件进行交互。当然，具体的框架内容要取决于框架的类型、Java 开发人员的技能水平、他们所要完成的工作以及自己的偏好。

为什么要使用框架

框架的优越性在于：

  1. 集中精力完成系统的业务逻辑设计；
  2. 使程序更容易扩展和维护，稳定性更强；
  3. 处理系统很多细节问题，比如安全性问题。

如何使用

  1. 导入相关的jar包
  2. 配置项目配置文件（比如web.xml），让项目用框架处理各种行为
  3. 添加框架配置文件（比如struts-config.xml），设定框架如何处理各种行为（比如接收到某个请求时，调用哪个类处理）
  4. 编写业务逻辑（继承、实现框架中的类、接口），完成具体处理操作。

主流框架

用得最多的Java三大框架是Struts、Spring和Hibernate ，简称SSH；14年以后逐步开始采用Spring、SpringMVC和MyBatis，简称SSM；而现在Java开发用得最多的框架其实是SpringBoot。

SSH的特点

SSH框架是以Struts框架进行MVC分离、控制业务跳转，同时使用Hibernate进行持久化，最后配合Spring的统一管理进行实现的开发框架。相对于传统的J2EE开发框架来讲性能相对较高、资源消耗相对较少。具体优势如下：

  • 框架进行了代码分层，将Java代码和HTML代码进行分层，大大降低了开发人员的技术难度，不需要同时精通Java后端知识和Web前端知识。
  • 由于使用了Struts框架进行了MVC分离，降低了各层代码间的耦合度。
  • 与Spring的AOP机制相互独立，互不影响。
  • 使用SSH框架开发项目，能够大大提高项目的开发效率与开发质量。
    良好的扩展性和维护性。

SSM框架的特点

SSM框架看着是Spring+SpringMVC+MyBatis三个框架整合而成的，其实SpringMVC属于Spring框架，所以相当于SSM框架就是Spring和Mybatis两个框架的整合运用

  • 对于MVC模式的实现，SpringMvc更加的轻量。
  • 由于SpringMvc本身属于Spring，因此与Spring的配合使用契合度更高。
  • Mybatis相对Hibernate更加轻量，同时将Sql和业务代码进行了很好的解耦。
  • MyBatis的学习门槛更低，相对于传统的JDBC差距不大，更好入门及掌握。

SpringBoot框架

SpringBoot框架是基于Spring4.0设计的，使用SpringBoot框架进行应用开发能够使用Spring框架所有优秀特性，同时还能够减少各种复杂的配置过程，降低各依赖包的冲突、增强系统的稳定性。我们可以将SpringBoot框架理解为我们使用SSM框架的一个脚手架框架使用，使用它能够非常方便的构建我们的应用。

  • 以非常方便和快速的构建Spring相关应用
  • 可以快速构建可执行的Jars和Wars文件
  • 内嵌了Servlet容器，如Tomcat、Jetty、Undertow
  • 使用SpringBoot框架开发项目，可以让Spring的配置操作几乎为零
  • SpringBoot框架提供了运行时监控功能
  • 框架极大的提高了项目开发和部署的效率
