有效的数独

发表于 2023-08-28

题目描述

请你判断一个 9 x 9 的数独是否有效。只需要 根据以下规则 ，验证已经填入的数字是否有效即可。

数字 1-9 在每一行只能出现一次。
数字 1-9 在每一列只能出现一次。
数字 1-9 在每一个以粗实线分隔的 3x3 宫内只能出现一次。（请参考示例图）

注意：

  • 一个有效的数独（部分已被填充）不一定是可解的。

  • 空白格用 ‘.’ 表示。

解决

方法一：遍历

用两个二维数组分别记录数独的每一行和每一列的每个数字的出现情况，再用一个三维数组记录每一个九宫格中每个数字出现的次数，如果有出现了超过两次的数据，就说明不满足有效数独，返回 false

题目描述

给定一个长度为 n 的整数数组 height 。有 n 条垂线，第 i 条线的两个端点是 (i, 0) 和 (i, height[i]) 。

找出其中的两条线，使得它们与 x 轴共同构成的容器可以容纳最多的水。

返回容器可以储存的最大水量。

说明：你不能倾斜容器。

方法一：双指针法

使用双指针分别指向数组的头尾，接下来的关键问题是：每次迭代如何移动指针。

首先对问题要求进行分析，要求出所能得到的最大容量，而每两根垂线之间的面积公式为：两者间长度乘以短板的高度。针对这个公式进行分析，

  • 每次移动较长线，两者间长度变小，新的短板的高度不变或变小，容量变小或不变
  • 每次移动较短线，两者间长度变小，新的短板的高度不变或变大，容量不变或变大

由此可知，每次迭代应该移动较短线。

多数元素

发表于 2023-08-28

题目描述

给定一个大小为 n 的数组 nums ，返回其中的多数元素。多数元素是指在数组中出现次数 大于 ⌊ n/2 ⌋ 的元素。

你可以假设数组是非空的，并且给定的数组总是存在多数元素。

解决

方法一：哈希法

我的第一反应是使用哈希表把每个数和对应的出现次数存放在哈希表中，当出现一个次数大于 ⌊ n/2 ⌋ 的元素时就将它返回。勉强完成了题目，但是时间不是很理想。虽然时间复杂度也达到了 O(n)，但是大量的哈希查询浪费了时间。

方法二：摩尔投票法

候选人(candNum)初始化为 num[0]，票数(count)初始化为 1，每当碰到与候选人相同的数字时票数加1，否则票数减1，票数为0时更换候选人，并将票数重置为1，遍历一次数组，结束时的 candNum 即为多数。

正确性：
多数元素的数量大于 ⌊ n/2 ⌋，也就是说，多数元素的个数减去其他元素的总和仍然 >= 1。

考虑 nums 的唯一元素的数量为 k ，你需要做以下事情确保你的题解可以被通过：

更改数组 nums ，使 nums 的前 k 个元素包含唯一元素，并按照它们最初在 nums 中出现的顺序排列。nums 的其余元素与 nums 的大小不重要。

判题标准:

如果所有断言都通过，那么您的题解将被通过。

解决

方法一：双指针法

注意到数组有序，重复的元素一定相邻。要求删除重复元素，只需将不重复的元素移到数组左侧即可。

考虑两个指针：left 和 right，每次迭代比较 left 和 right 指针的元素是否相等：如果不相等，将 right 指针的元素复制到 left 上，然后 left 右移一位；无论是否相等，均在迭代的最后将 right 右移一位。直至 right 到达数组末尾。此时 left+1 即为新数组的长度。

判断子序列

发表于 2023-08-25 更新于 2023-08-28

题目描述

给定字符串 s 和 t ，判断 s 是否为 t 的子序列。

字符串的一个子序列是原始字符串删除一些（也可以不删除）字符而不改变剩余字符相对位置形成的新字符串。（例如，”ace”是”abcde”的一个子序列，而”aec”不是）。

进阶：

如果有大量输入的 S，称作 S1, S2, … , Sk 其中 k >= 10亿，你需要依次检查它们是否为 T 的子序列。在这种情况下，你会怎样改变代码？

解决

方法一：双指针法

这道题可以使用贪心法：题目要求“一个子序列是原始字符串删除一些（也可以不删除）字符而不改变剩余字符相对位置形成的新字符串”，因此只要子序列是原始字符串的一部分就行，而不要求字符对应的完整次序，比如”ace”是”asfcceef”的子序列，而不用考虑”ace”中的’c’和’e’对应原始字符串的哪个’c’和’e’。

为了判断这个，使用贪心法：用两个指针分别指向待判断字符串和原始字符串的起始，然后从前往后依次匹配即可。贪心策略的正确性证明在这里忽略不写。

方法二：动态规划

考虑前面的双指针的做法，我们注意到我们有大量的时间用于在 t 中找到下一个匹配字符。

这样我们可以预处理出对于 t 的每一个位置，从该位置开始往后每一个字符第一次出现的位置。

我们可以使用动态规划的方法实现预处理，令 f[i][j] 表示字符串 t 中从位置 i 开始往后字符 j 第一次出现的位置。在进行状态转移时，如果 t 中位置 i 的字符就是 j，那么 f[i][j]=i，否则 j 出现在位置 i+1 开始往后，即 f[i][j]=f[i+1][j]，因此我们要倒过来进行动态规划，从后往前枚举 i

移除元素

发表于 2023-08-25 更新于 2023-08-28

题目描述

给你一个数组 nums 和一个值 val，你需要 原地 移除所有数值等于 val 的元素，并返回移除后数组的新长度。

不要使用额外的数组空间，你必须仅使用 O(1) 额外空间并 原地 修改输入数组。

元素的顺序可以改变。你不需要考虑数组中超出新长度后面的元素。

解决

方法一：双指针法

使用双指针法，分别指向数组头和尾，每次头指针指向 val 值时，令头指针所指变成尾指针的值，如果仍为 val 值，则将尾指针向前移动一位，继续令头指针所指变成尾指针的值，直到与 val 值不相等。在头尾指针相遇之后结束循环，此时头指针的大小即为新数组的长度。

题目描述

给定一个含有 n 个正整数的数组和一个正整数 target 。

找出该数组中满足其和 ≥ target 的长度最小的 连续子数组 [numsl, numsl+1, …, numsr-1, numsr] ，并返回其长度。如果不存在符合条件的子数组，返回 0。

解决

方法一：二分查找法

首先计算前缀和数组sums，然后对于每个开始下标 i，可通过二分查找得到大于或等于 i 的最小下标 temp，使得 sums[temp]-sums[i-1]≥s，并更新子数组的最小长度

方法二：滑动窗口

使用方法一时，每次确定满足条件的最小下标 temp 耗费了大量的时间，为了对这点进行优化，使用滑动窗口。

定义两个指针 start 和 end 表示滑动窗口(子数组)的开始和结束位置，并用 sum 存储子数组元素和。

每轮迭代，将 nums[end] 加到 sum 上，如果 sum ≥ s，则去更新子数组的最小长度，然后将 nums[start] 从 sum 减去并将 start 右移，直到 sum < s，同时更新子数组的最小长度。迭代结束时，将 end 右移。
